**外部中断** 是一种**硬件中断机制**，用于处理外部设备或信号源的异步事件。在嵌入式系统或微控制器中，外部中断允许处理器停止当前的指令执行，并及时响应外部输入信号。外部中断通常来自外部设备，比如按钮、传感器、通信设备等。它能够打断正常程序的执行，并引发一个预定义的中断处理函数（ISR, Interrupt Service Routine）来处理这些事件。

**外部中断与內部中断的区别**

外部中断与內部中断它们的区别主要在于**中断的触发源**不同：

外部中断：

- 由外部事件或信号触发，通常是**通过微控制器的外部引脚接收到的信号**。例如，按钮按下、传感器信号变化、通信数据到达等都会触发外部中断。
- 典型来源包括：外部输入引脚（如 GPIO 引脚）、传感器、外部设备等。

内部中断：

- **由微控制器或系统内部事件触发**。例如，定时器溢出、ADC 转换完成、串口数据发送完成等。
- 典型来源包括：定时器、计数器、内部通信模块（如 UART、SPI）、模拟数字转换器（ADC）等。

### 实现

```c
// main.c
#include "Exti.h"
#include "NVIC.h"

/******************** INT配置 ********************/
void	Exti_config(void)
{
    EXTI_InitTypeDef	Exti_InitStructure;							//结构定义

    Exti_InitStructure.EXTI_Mode      = EXT_MODE_RiseFall;//中断模式,   EXT_MODE_RiseFall,EXT_MODE_Fall
    Ext_Inilize(EXT_INT0,&Exti_InitStructure);				//初始化
    NVIC_INT0_Init(ENABLE,Priority_0);		//中断使能, ENABLE/DISABLE; 优先级(低到高) Priority_0,Priority_1,Priority_2,Priority_3
}
void ext_int0_call(void) {
	// 当中断触发时的实现逻辑
}
=================================================================================
// Exit_lsr.c
extern void ext_int0_call();
//========================================================================
// 函数: INT0_ISR_Handler
// 描述: INT0中断函数.
// 参数: none.
// 返回: none.
// 版本: V1.0, 2020-09-23
//========================================================================
void INT0_ISR_Handler (void) interrupt INT0_VECTOR		//进中断时已经清除标志
{
	ext_int0_call();
}
```
